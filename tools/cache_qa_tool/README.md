# Cache QA Eval Tool

A standalone visual tool for QA to evaluate WebA11y alt-text **cache validation**.

Each row shows:
- 4 images side-by-side (served base, populator base, served context, populator context)
- Both website URLs
- Cached alt-text vs existing alt-text
- Cache key + ETags

QA labels each row with two fields:
1. **Bucket** (pair similarity): B1 / B2 / B3 / NA
2. **Alt-text accuracy** (does the cached alt actually describe the served image?): correct / incorrect / partial / NA
3. Optional notes

The tool exports a CSV ready for analysis.

---

## How to run (no install needed)

1. Open `index.html` in **any modern browser** (Chrome / Safari / Firefox):

   ```bash
   open /Users/tusharpandole/Downloads/cache_qa_tool/index.html
   ```

   Or just double-click the file.

2. Click **"Click to upload CSV"** and select `qa_hits_only.csv` (generated by the pipeline).

3. The tool **automatically filters** to only show rows where BOTH served + populator info are present. Rows with missing populator data are excluded.

4. For each row:
   - Look at the 4 images (click any to enlarge).
   - Compare website URLs (auto-tagged SAME / DIFF).
   - Read existing vs cached alt text.
   - Pick **bucket** + **alt-text accuracy** + optional notes.

5. When done, click **💾 Export CSV** at top right to download the labeled CSV.

## Buckets explained

| Bucket | Definition | Cache FP risk |
|---|---|---|
| **B1** | Same base image, **different website** | High — context drastically different |
| **B2** | Same site, **different context** image | Medium — image used in different page section |
| **B3** | Same everything (TP) | Low — cache hit fully justified |
| **NA** | Cannot determine (image missing/unclear) | — |

## Alt accuracy explained

| Label | Meaning |
|---|---|
| **correct** | Alt text accurately describes the served image |
| **incorrect** | Wrong / hallucinated / misleading for served image |
| **partial** | Mostly OK but missing important context info |
| **NA** | Cannot determine (image missing, alt missing, etc.) |

## Why both labels?

- **Bucket** measures structural/pair correctness (was the cache hit *defensible*?)
- **Alt accuracy** measures actual functional correctness (is the alt text useful for an end user?)

A bucket-3 (TP) cache hit can still propagate a *bad* alt text if the original LLM output was wrong. A bucket-1 (cross-site) cache hit can sometimes still produce a *correct* alt text (image content is what matters, not page context).

The combination (`bucket × alt_accuracy`) tells you the **real cache-induced FP rate**:
- B3 + correct → cache working perfectly ✅
- B1/B2 + incorrect → cache-induced FP ⚠️ (the metric you care about)

## Auto-save

Annotations are auto-saved to your browser's localStorage. You can close the tab and resume — your labels persist. Use the **Reset** button to clear all annotations if needed.

To move work to another browser/machine, click **Export CSV** to download a snapshot.

---

## Tips for QA

- **Filter focus**: only rows with full populator info appear. Missing-populator rows are excluded automatically.
- **Trust SAME/DIFF badges**: the tool pre-computes whether website / base ETag / context ETag match. Use them to short-cut bucketing.
- **B3 + Incorrect is the most important signal** — it means an existing LLM hallucination is being amplified by the cache.
- **B1/B2 + Incorrect is the cache-induced FP** — alt text was right for the populator's image but wrong for the served one.
- Use **Jump to next unlabeled** to skip already-labeled rows.
